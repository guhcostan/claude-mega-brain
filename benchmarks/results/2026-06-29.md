## Method

Two prompts, same model:
- **baseline**: plain question, no context
- **mega-brain**: question prefixed with the project's OKF data dictionary (simulates `SessionStart` hook injection)

6 questions about a fictional schema with column names unknowable from training data (`amount_cents`, `refund_cents`, `acme.com` exclusion filter, `done` status, recent changelog entry). A correct answer requires the injected context.

## Interpretation

Questions 1–3 and 5 are unanswerable without the OKF context — baseline scores 0/5 on all.
Questions 4 and 6 have answers partially guessable from training data (`done`, `customer_id`); baseline scores partial marks but mega-brain is still 100%.

## Limitations

- Single-turn Q&A, not agentic sessions — doesn't measure tool call reduction
- Synthetic schema, not production data
- n=5 per cell is small; run n=20 for publication
- `done` and `customer_id` tests partially overlap with training data
